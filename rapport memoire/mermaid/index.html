Figure 3.6 — États du cycle de vie mission

stateDiagram-v2
    [*] --> pending
    pending --> funded : Paiement confirmé
    pending --> cancelled
    funded --> accepted : Candidature acceptée
    funded --> cancelled
    accepted --> in_progress : Caution + start
    accepted --> funded : Délai 4h expiré
    in_progress --> submitted : Preuves soumises
    in_progress --> disputed
    submitted --> completed : Validation client
    submitted --> disputed
    completed --> [*]
    cancelled --> [*]

Figure 3.7 — Séquence création mission

sequenceDiagram
    actor Client
    participant FE as Angular
    participant API as Django REST
    participant MM as Mobile Money
    participant Meta as MetaMask
    participant SC as EscrowContract
    Client->>FE: Crée mission
    FE->>API: POST /missions/
    API-->>FE: status=pending
    Client->>FE: Paiement OTP
    FE->>API: confirm_payment
    API->>MM: process_payment
    MM-->>API: OK
    API-->>FE: status=funded
    opt Blockchain optionnel
        FE->>Meta: createMission
        Meta->>SC: tx
        FE->>API: record-mission
    end
